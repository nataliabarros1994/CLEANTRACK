# Contributing to CleanTrack

Thank you for your interest in contributing to CleanTrack!

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch
4. Make your changes
5. Run tests
6. Submit a pull request

## Code Style

We follow PEP 8 for Python code style. Please ensure your code:

- Is properly formatted
- Includes docstrings for functions and classes
- Has meaningful variable names
- Follows Django best practices

## Commit Messages

Use clear and descriptive commit messages:

```
feat: Add IoT webhook endpoint for cleaning logs
fix: Resolve timezone issue in compliance alerts
docs: Update README with deployment instructions
test: Add tests for Equipment model
```

## Pull Request Process

1. Update the README.md with details of changes if needed
2. Update the documentation
3. Ensure all tests pass
4. Request review from maintainers

## Testing

Run the test suite before submitting:

```bash
python manage.py test
```

## Questions?

Contact the development team at dev@cleantrack.app
